## 일반

<Accordion title="어떤 HWP 버전을 지원하나요?">
HWP v5.0 이상을 지원해요. 아래한글 2002 이후 버전에서 생성된 대부분의 `.hwp` 파일이 호환돼요.

HWPX는 아래한글 2014 이후 버전의 `.hwpx` 파일을 지원해요.

**확인 방법**: 한컴오피스에서 파일을 열고 **도움말 → 제품 정보**에서 버전을 확인하세요.
</Accordion>

<Accordion title="이미지는 어떻게 처리되나요?">
현재 버전(v0.0.x)에서는 이미지 추출을 지원하지 않아요. 대신 다음과 같이 처리돼요:

- **HWP/HWPX**: 이미지 위치에 `[Image]` 텍스트로 표시
- **PDF**: 이미지는 무시되고 텍스트만 추출

이미지 추출 기능은 향후 버전(v0.1.0+)에서 지원 예정이에요.

**대안**: 이미지가 중요한 문서는 한컴오피스에서 직접 PDF로 변환 후 사용하세요.
</Accordion>

<Accordion title="다른 파일 포맷도 지원하나요?">
현재 지원하는 포맷은 다음과 같아요:

- **입력**: `.hwp`, `.hwpx`, `.pdf` (선택적)
- **출력**: Markdown (`.md`)

향후 추가 예정 포맷:
- **입력**: DOCX, ODT, RTF
- **출력**: HTML, reStructuredText, AsciiDoc

커스텀 포맷이 필요하다면 [포맷 확장 가이드](/extending)를 참고해 직접 Parser/Writer를 추가할 수 있어요.
</Accordion>

<Accordion title="상업적으로 사용해도 되나요?">
네! Ureca Document Parser는 MIT 라이선스로 배포되므로 상업적 사용이 자유로워요.

단, 다음 의존성의 라이선스도 확인하세요:
- `olefile`: BSD License
- `pymupdf`: AGPL License (상업적 사용 시 별도 라이선스 필요)

PDF 기능을 상업적으로 사용하려면 PyMuPDF 라이선스를 확인하거나 대체 라이브러리(pdfplumber 등)를 사용하세요.
</Accordion>

## 설치 및 환경

<Accordion title="Python 3.11에서 설치가 안 돼요">
이 라이브러리는 Python 3.12 이상이 필요해요. Python 버전을 업그레이드하세요:

```bash
# pyenv 사용 시
pyenv install 3.12.0
pyenv global 3.12.0

# 또는 시스템 패키지 관리자
sudo apt install python3.12  # Ubuntu/Debian
brew install python@3.12     # macOS
```

**이유**: Python 3.12의 타입 힌팅 기능(`type` statement)을 사용하기 때문이에요.
</Accordion>

<Accordion title="LangChain 설치 없이 사용할 수 있나요?">
네! 기본 설치에는 LangChain이 포함되지 않아요.

```bash
# LangChain 없이 설치
pip install ureca_document_parser

# Markdown 변환만 사용
from ureca_document_parser import convert
convert("document.hwp", "output.md")  # ✅ 작동

# 청크 변환 시도 시
convert("document.hwp", chunks=True)  # ❌ ImportError
```

LangChain 통합이 필요하면 `[langchain]` extra를 설치하세요:

```bash
pip install ureca_document_parser[langchain]
```
</Accordion>

<Accordion title="M1/M2 Mac에서 설치 오류가 나요">
Apple Silicon Mac에서 `olefile` 설치 시 문제가 발생할 수 있어요. Rosetta 없이 네이티브로 설치하세요:

```bash
# Homebrew로 Python 설치 (네이티브 ARM64)
brew install python@3.12

# 설치
pip3.12 install ureca_document_parser
```

**디버깅**: `arch` 명령어로 현재 아키텍처를 확인하세요:

```bash
arch  # arm64 출력 확인
```
</Accordion>

## 사용법

<Accordion title="CLI에서 여러 파일을 한 번에 변환하려면?">
Shell 루프를 사용하세요:

```bash
# 현재 디렉토리의 모든 HWP 파일 변환
for file in *.hwp; do
    uv run ureca_document_parser "$file" -o "${file%.hwp}.md"
done

# 하위 디렉토리 포함
find . -name "*.hwp" -exec uv run ureca_document_parser {} -o {}.md \;
```

Python에서는 `pathlib.Path.glob()`을 사용하세요:

```python
from pathlib import Path
from ureca_document_parser import convert

for hwp_file in Path(".").glob("**/*.hwp"):
    md_file = hwp_file.with_suffix(".md")
    convert(hwp_file, md_file)
```
</Accordion>

<Accordion title="표가 제대로 변환되지 않아요">
복잡한 표 구조는 완벽하게 변환되지 않을 수 있어요. 현재 제한사항:

- ❌ **병합된 셀**: 일부 병합 패턴 미지원
- ❌ **중첩 표**: HTML로 대체 변환됨
- ❌ **셀 배경색/테두리**: 스타일 정보 손실

**해결 방법**:
1. 한컴오피스에서 표를 단순화 (병합 해제)
2. Markdown 출력 후 수동으로 표 수정
3. HTML Writer를 사용 (향후 지원 예정)

**디버깅**: 고급 모드로 Document 모델을 직접 확인하세요:

```python
from ureca_document_parser import get_registry

registry = get_registry()
doc = registry.parse("document.hwp")

# 표 구조 확인
for elem in doc.elements:
    if isinstance(elem, Table):
        print(f"Table with {len(elem.rows)} rows")
        for row in elem.rows:
            print(f"  Row with {len(row.cells)} cells")
```
</Accordion>

<Accordion title="한글 파일명이 깨져요 (Windows)">
Windows에서 한글 경로 처리 시 `Path` 객체를 사용하세요:

```python
from pathlib import Path

# ❌ 문자열 직접 사용 (깨질 수 있음)
convert("문서.hwp", "출력.md")

# ✅ Path 객체 사용 (권장)
convert(Path("문서.hwp"), Path("출력.md"))
```

**CLI에서는** 따옴표로 감싸세요:

```bash
# Windows PowerShell
uv run ureca_document_parser "문서.hwp" -o "출력.md"
```
</Accordion>

<Accordion title="대용량 파일 처리 시 메모리 부족 오류가 나요">
현재 버전은 파일 전체를 메모리에 로드하므로, 100MB 이상의 대용량 문서는 메모리 부족이 발생할 수 있어요.

**해결 방법**:
1. **파일 분할**: 한컴오피스에서 문서를 여러 파일로 분할
2. **메모리 증설**: 시스템 RAM 증가
3. **스트리밍 모드**: 향후 버전에서 지원 예정

**현재 메모리 사용량 추정**:
- HWP 파일: 원본 크기의 약 3~5배
- HWPX 파일: 원본 크기의 약 2~3배
- PDF 파일: 페이지 수 × 1MB 정도
</Accordion>

## LangChain 통합

<Accordion title="RAG 파이프라인에서 청크 크기는 어떻게 설정하나요?">
문서 종류와 용도에 따라 다르지만, 일반적인 권장사항은 다음과 같아요:

```python
from ureca_document_parser import convert

# 짧은 문서 (보고서, 공문)
chunks = convert("보고서.hwp", chunks=True, chunk_size=500, chunk_overlap=100)

# 긴 문서 (논문, 매뉴얼)
chunks = convert("매뉴얼.hwp", chunks=True, chunk_size=1500, chunk_overlap=200)

# 질의응답 최적화 (더 큰 컨텍스트)
chunks = convert("문서.hwp", chunks=True, chunk_size=2000, chunk_overlap=400)
```

**일반 원칙**:
- `chunk_size`: LLM 입력 토큰의 20~30%
- `chunk_overlap`: `chunk_size`의 10~20%
- 문단 단위로 자르려면 `chunk_size`를 크게 설정
</Accordion>

<Accordion title="벡터 스토어에 저장할 때 메타데이터를 추가하려면?">
`convert()`로 생성된 청크는 기본 메타데이터(`source`, `format`)만 포함해요. 추가 메타데이터는 수동으로 설정하세요:

```python
from ureca_document_parser import convert
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings

chunks = convert("보고서.hwp", chunks=True)

# 메타데이터 추가
for chunk in chunks:
    chunk.metadata["department"] = "연구개발팀"
    chunk.metadata["year"] = 2024
    chunk.metadata["category"] = "기술문서"

# 벡터 스토어에 저장
embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(chunks, embeddings)

# 메타데이터 필터링
results = vectorstore.similarity_search(
    "프로젝트 일정",
    filter={"department": "연구개발팀"}
)
```
</Accordion>

## 트러블슈팅

<Accordion title="'지원하지 않는 파일 형식입니다' 오류가 나요">
파일 확장자가 정확한지 확인하세요:

```bash
# 파일 확장자 확인
file document.hwp  # "Hangul (Korean) Word Processor File" 출력 확인

# 지원 포맷 확인
uv run ureca_document_parser --list-formats
```

**일반적인 원인**:
- 확장자 대소문자 (`Document.HWP` → `.hwp`로 변경)
- 잘못된 확장자 (`.doc` 파일을 `.hwp`로 이름 변경)
- 손상된 파일 (한컴오피스에서 다시 저장)
</Accordion>

<Accordion title="'ParseError: Failed to parse' 오류가 나요">
파일이 손상되었거나 지원하지 않는 HWP 버전일 수 있어요.

**디버깅 단계**:

1. **한컴오피스에서 파일 열기 테스트**:
   - 열리지 않으면 → 파일 손상
   - 열리면 → 버전 확인 (v5.0 미만은 미지원)

2. **다시 저장하기**:
   ```
   한컴오피스 열기 → 다른 이름으로 저장 → HWP 5.0 포맷 선택
   ```

3. **HWPX로 변환 시도**:
   ```
   한컴오피스 열기 → 다른 이름으로 저장 → HWPX 포맷 선택
   ```

4. **이슈 제기**:
   파일을 [GitHub Issues](https://github.com/ureca-corp/document_parser/issues)에 첨부해 주세요 (민감한 정보 제거 후).
</Accordion>

<Accordion title="텍스트 일부가 누락돼요">
HWP/HWPX 파일의 복잡한 레이아웃은 완벽하게 추출되지 않을 수 있어요.

**알려진 제한사항**:
- ❌ 텍스트 박스 내용
- ❌ 머리글/바닥글
- ❌ 각주/미주
- ❌ 댓글/메모

**확인 방법**: Document 모델을 직접 출력해 보세요:

```python
from ureca_document_parser import get_registry

registry = get_registry()
doc = registry.parse("document.hwp")

# 모든 텍스트 출력
for elem in doc.elements:
    if hasattr(elem, 'text'):
        print(elem.text)
```

누락된 텍스트가 중요하다면 한컴오피스에서 PDF로 변환 후 PDF Parser를 사용하세요.
</Accordion>

## 성능

<Accordion title="변환 속도가 느려요">
파일 크기와 포맷에 따라 속도가 달라요:

**일반적인 처리 시간** (단일 코어 기준):
- HWP 10MB: ~500ms
- HWPX 10MB: ~1~2초 (XML 파싱 오버헤드)
- PDF 100페이지: ~3~5초

**속도 개선 팁**:

1. **병렬 처리**: 여러 파일을 동시에 변환
   ```python
   from concurrent.futures import ThreadPoolExecutor
   from ureca_document_parser import convert
   
   files = ["doc1.hwp", "doc2.hwp", "doc3.hwp"]
   
   with ThreadPoolExecutor(max_workers=4) as executor:
       executor.map(lambda f: convert(f, f"{f}.md"), files)
   ```

2. **HWPX → HWP 변환**: HWPX가 느리다면 HWP로 저장 후 처리

3. **불필요한 요소 제거**: 한컴오피스에서 이미지/표 제거 후 변환
</Accordion>

## 기여

<Accordion title="버그를 발견했어요. 어떻게 리포트하나요?">
[GitHub Issues](https://github.com/ureca-corp/document_parser/issues)에 다음 정보와 함께 이슈를 생성해 주세요:

**환경**:
- Python 버전: 3.12.0
- 패키지 버전: 0.0.9
- OS: macOS 14.0

**재현 방법**:
1. `document.hwp` 파일 준비 (첨부)
2. 다음 코드 실행:
   ```python
   from ureca_document_parser import convert
   convert("document.hwp", "output.md")
   ```

**기대 동작**: 표가 Markdown으로 변환되어야 함

**실제 동작**: `ParseError: Failed to parse table` 발생

**에러 메시지**: 전체 traceback 첨부

민감한 정보가 포함된 파일은 샘플 데이터로 대체해 주세요.
</Accordion>

[GitHub Discussions](https://github.com/ureca-corp/document_parser/discussions)에 기능 제안을 올려주세요.

포함하면 좋은 내용:
- **배경**: 왜 이 기능이 필요한가요?
- **제안 내용**: 어떤 API/기능을 원하시나요?
- **사용 예시**: 코드 예제
- **대안**: 현재 어떻게 우회하고 계신가요?

**예시**:
- 제안: DOCX 포맷 지원
- 배경: MS Word 문서를 많이 다루는데, 일일이 HWP로 변환하는 게 번거로워요.
- 현재 우회 방법: python-docx로 직접 파싱 후 Markdown 변환 중
</Accordion>
