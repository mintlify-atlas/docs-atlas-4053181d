## 개요

`ureca_document_parser`는 Python API뿐만 아니라 커맨드라인 인터페이스(CLI)도 제공해요. 스크립트나 쉘에서 빠르게 문서를 변환할 수 있어요.

## 기본 사용법

CLI는 `ureca_document_parser` 명령어로 실행해요.

```bash
# uv 환경에서 실행
uv run ureca_document_parser <입력파일> [옵션]

# 전역 설치된 경우
ureca_document_parser <입력파일> [옵션]
```

<Info>
이 가이드의 모든 예제는 `uv run` 환경을 기준으로 작성되었어요. 전역 설치 시에는 `uv run` 부분을 생략하면 돼요.
</Info>

### 파일로 저장하기

`-o` 또는 `--output` 옵션으로 출력 파일을 지정해요.

```bash
uv run ureca_document_parser 보고서.hwp -o 보고서.md
```

**출력 예시:**
```
변환 완료: 보고서.md
```

### 표준 출력으로 출력

출력 파일을 지정하지 않으면 변환 결과가 표준 출력(stdout)으로 출력돼요.

```bash
uv run ureca_document_parser 보고서.hwp
```

**출력 예시:**
```markdown
# 프로젝트 보고서

## 개요

이 문서는 프로젝트의 진행 상황을 정리한 보고서입니다.

## 주요 내용

- 목표 달성률: 85%
- 예산 사용률: 72%
...
```

### 리다이렉션으로 파일 저장

쉘 리다이렉션을 사용해서 파일로 저장할 수도 있어요.

```bash
uv run ureca_document_parser 보고서.hwp > 보고서.md
```

## 옵션

### `-o, --output`

출력 파일 경로를 지정해요.

```bash
uv run ureca_document_parser 제안서.hwpx -o output/제안서.md
```

출력 디렉토리가 없으면 자동으로 생성돼요.

### `-f, --format`

출력 포맷을 지정해요. 기본값은 `markdown`이에요.

```bash
uv run ureca_document_parser 문서.hwp -f markdown -o 출력.md
```

<Note>
현재는 `markdown` 포맷만 지원해요.
</Note>

### `--list-formats`

지원하는 입력/출력 형식을 출력해요.

```bash
uv run ureca_document_parser --list-formats
```

**출력 예시:**
```
지원 입력 형식: .hwp, .hwpx
지원 출력 형식: markdown
```

### `-h, --help`

도움말을 출력해요.

```bash
uv run ureca_document_parser --help
```

**출력 예시:**
```
usage: ureca_document_parser [-h] [--output OUTPUT] [--format FORMAT]
                              [--list-formats]
                              [input_file]

문서 파일을 다양한 형식으로 변환합니다.

positional arguments:
  input_file            변환할 입력 파일 경로

optional arguments:
  -h, --help            show this help message and exit
  --output OUTPUT, -o OUTPUT
                        출력 파일 경로 (미지정 시 stdout 출력)
  --format FORMAT, -f FORMAT
                        출력 형식 (기본값: markdown)
  --list-formats        지원하는 입력/출력 형식을 출력합니다
```

## 실전 예제

### 여러 파일 일괄 변환

<CodeGroup>

```bash Bash (for 루프)
for file in *.hwp; do
    uv run ureca_document_parser "$file" -o "${file%.hwp}.md"
done

for file in *.hwpx; do
    uv run ureca_document_parser "$file" -o "${file%.hwpx}.md"
done
```

```bash Bash (find + xargs)
find documents -name "*.hwp" -print0 | while IFS= read -r -d '' file; do
    output="output/$(basename "${file%.hwp}.md")"
    uv run ureca_document_parser "$file" -o "$output"
done
```

```bash 진행률 표시
total=$(ls -1 *.hwp *.hwpx 2>/dev/null | wc -l)
current=0

for file in *.hwp *.hwpx; do
    [ -f "$file" ] || continue
    current=$((current + 1))
    echo "[$current/$total] 변환 중: $file"
    uv run ureca_document_parser "$file" -o "${file%.*}.md"
done

echo "✓ 변환 완료: $total개 파일"
```

</CodeGroup>

### 디렉토리 구조 유지하며 변환

입력 디렉토리의 구조를 그대로 유지하면서 변환하는 예제예요.

```bash
#!/bin/bash

input_dir="documents"
output_dir="output"

find "$input_dir" -type f \( -name "*.hwp" -o -name "*.hwpx" \) | while read file; do
    # 상대 경로 계산
    rel_path="${file#$input_dir/}"
    output_file="$output_dir/${rel_path%.*}.md"
    
    # 출력 디렉토리 생성
    mkdir -p "$(dirname "$output_file")"
    
    # 변환 실행
    uv run ureca_document_parser "$file" -o "$output_file"
    echo "✓ $rel_path"
done
```

### 에러 처리 포함

변환 실패 시 에러를 처리하는 예제예요.

```bash
#!/bin/bash

success=0
failure=0

for file in *.hwp *.hwpx; do
    [ -f "$file" ] || continue
    
    if uv run ureca_document_parser "$file" -o "${file%.*}.md" 2>/dev/null; then
        echo "✓ $file"
        success=$((success + 1))
    else
        echo "✗ $file (변환 실패)"
        failure=$((failure + 1))
    fi
done

echo ""
echo "변환 완료: $success개"
echo "변환 실패: $failure개"
```

### 파이프와 함께 사용

표준 출력을 다른 명령어와 연결할 수 있어요.

```bash
# 단어 수 세기
uv run ureca_document_parser 보고서.hwp | wc -w

# 특정 단어 검색
uv run ureca_document_parser 보고서.hwp | grep -i "예산"

# 첫 10줄만 출력
uv run ureca_document_parser 보고서.hwp | head -10

# 파일 크기 확인
uv run ureca_document_parser 보고서.hwp | wc -c
```

## 에러 메시지

CLI에서 발생할 수 있는 주요 에러 메시지예요.

### 파일을 찾을 수 없음

```bash
uv run ureca_document_parser 없는파일.hwp -o 출력.md
```

**출력:**
```
오류: 파일을 찾을 수 없습니다: 없는파일.hwp
```

**종료 코드:** `1`

### 지원하지 않는 파일 형식

```bash
uv run ureca_document_parser 문서.docx -o 출력.md
```

**출력:**
```
오류: 지원하지 않는 파일 형식입니다: .docx (지원: .hwp, .hwpx)
```

**종료 코드:** `1`

### 파싱 실패

```bash
uv run ureca_document_parser 손상된파일.hwp -o 출력.md
```

**출력:**
```
오류: <파싱 에러 메시지>
```

**종료 코드:** `1`

## 내부 동작

CLI는 다음과 같은 순서로 동작해요:

<Steps>
  <Step title="인자 파싱">
    `argparse`를 사용해서 커맨드라인 인자를 파싱해요.
  </Step>
  
  <Step title="파일 존재 확인">
    입력 파일이 존재하는지 확인해요. 없으면 에러 메시지를 출력하고 종료해요.
  </Step>
  
  <Step title="레지스트리에서 파서 선택">
    파일 확장자를 보고 적절한 파서를 자동으로 선택해요 (`registry.parse()`)
  </Step>
  
  <Step title="문서 파싱">
    선택된 파서로 문서를 `Document` 모델로 파싱해요.
  </Step>
  
  <Step title="Writer로 변환">
    `Document` 모델을 지정된 포맷(기본값: markdown)으로 변환해요 (`registry.write()`)
  </Step>
  
  <Step title="결과 출력">
    출력 파일이 지정되었으면 파일로 저장하고, 아니면 표준 출력으로 출력해요.
  </Step>
</Steps>

<Info>
소스 코드는 `src/ureca_document_parser/cli.py`에서 확인할 수 있어요.
</Info>

## 종료 코드

CLI는 다음과 같은 종료 코드를 반환해요:

| 코드 | 의미 |
|------|------|
| `0` | 성공 |
| `1` | 에러 발생 (파일 없음, 파싱 실패, 지원하지 않는 형식 등) |

쉘 스크립트에서 종료 코드를 확인할 수 있어요:

```bash
uv run ureca_document_parser 보고서.hwp -o 보고서.md
if [ $? -eq 0 ]; then
    echo "변환 성공"
else
    echo "변환 실패"
fi
```
