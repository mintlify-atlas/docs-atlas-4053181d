# HwpParser

HWP v5 바이너리 포맷을 파싱하여 `Document` 모델로 변환하는 파서예요.

## 개요

`HwpParser`는 한글과컴퓨터의 "HWP Binary Specification 1.1"을 따르는 OLE2 기반 HWP 바이너리 파일을 처리해요. 문서 구조를 분석하여 문단, 표, 제목 레벨 정보를 추출합니다.

### 파싱 파이프라인

```
OLE2 파일 → 섹션 압축 해제 → 바이너리 레코드 파싱 → 요소 추출 → Document
```

## 클래스 메서드

### `extensions()`

지원하는 파일 확장자 목록을 반환해요.

```python
@staticmethod
def extensions() -> list[str]
```

<ResponseField name="return" type="list[str]">
  지원하는 확장자 리스트. HWP 파서의 경우 `[".hwp"]`를 반환해요.
</ResponseField>

**사용 예시**

```python
from ureca_document_parser.hwp import HwpParser

print(HwpParser.extensions())  # [".hwp"]
```

### `parse()`

HWP 파일을 파싱하여 `Document` 객체를 반환해요.

```python
@staticmethod
def parse(filepath: Path | str) -> Document
```

<ParamField path="filepath" type="Path | str" required>
  파싱할 HWP 파일의 경로. 문자열 또는 `pathlib.Path` 객체를 받아요.
</ParamField>

<ResponseField name="return" type="Document">
  파싱된 문서 객체. `elements` 리스트에 `Paragraph`, `Table` 등의 문서 요소가 포함돼요.
</ResponseField>

**예외**

- `ParseError` — 파일을 찾을 수 없거나 유효한 HWP 파일이 아닌 경우 발생해요.

**사용 예시**

```python
from ureca_document_parser.hwp import HwpParser
from pathlib import Path

# 문자열 경로 사용
doc = HwpParser.parse("보고서.hwp")

# Path 객체 사용
path = Path("제안서.hwp")
doc = HwpParser.parse(path)

# 문서 요소 접근
for element in doc.elements:
    if isinstance(element, Paragraph):
        print(element.text)
```

## Parser Protocol 구현

`HwpParser`는 `Parser` 프로토콜을 구현해요. 상속 없이 정적 메서드 시그니처만 맞추면 되는 구조적 서브타이핑(structural subtyping) 방식이에요.

```python
class Parser(Protocol):
    @staticmethod
    def extensions() -> list[str]:
        """지원하는 파일 확장자 리스트"""
        ...

    @staticmethod
    def parse(filepath: Path | str) -> Document:
        """파일을 파싱하여 Document 반환"""
        ...
```

## 레지스트리 등록

`HwpParser`는 자동으로 `FormatRegistry`에 등록돼요. 수동 등록이 필요한 경우:

```python
from ureca_document_parser import get_registry
from ureca_document_parser.hwp import HwpParser

registry = get_registry()
registry.register_parser(HwpParser)
```

등록 후에는 확장자 기반 자동 라우팅이 가능해요:

```python
from ureca_document_parser import get_registry

registry = get_registry()
doc = registry.parse("문서.hwp")  # HwpParser가 자동으로 선택됨
```

## 내부 구현 세부사항

### 압축 처리

HWP 문서는 `FileHeader` 스트림의 플래그를 확인하여 압축 여부를 판단해요. 압축된 경우 zlib deflate 알고리즘으로 압축 해제합니다.

### 스타일 기반 제목 추출

`DocInfo` 스트림에서 스타일 테이블을 읽어 각 문단의 `style_id`를 제목 레벨(`heading_level`)로 매핑해요. "제목 1", "Heading 1" 등의 스타일 이름 패턴을 인식합니다.

### 섹션 헤더 테이블 변환

한국 공공기관 문서에서 흔히 사용하는 섹션 헤더 테이블 패턴을 감지해요:

```
| 번호 | 빈칸 | 제목 | 빈칸 |
```

이 패턴을 만족하는 2행 이하 테이블은 자동으로 `heading_level=2`인 `Paragraph`로 변환돼요.

### 폴백 메커니즘

`BodyText/Section*` 스트림에서 추출에 실패하면 `PrvText` 스트림(미리보기 텍스트)을 대체 소스로 사용해요.
