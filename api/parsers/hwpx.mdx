# HwpxParser

HWPX 포맷을 파싱하여 `Document` 모델로 변환하는 파서예요.

## 개요

`HwpxParser`는 ZIP 기반 Open Packaging Convention(OPC) 포맷인 HWPX 파일을 처리해요. ZIP 아카이브 내부의 XML 파일에서 문서 구조를 읽어 텍스트, 표, 제목 정보를 추출합니다.

### 파싱 파이프라인

```
ZIP 아카이브 → 섹션 XML 탐색 → XML 파싱 → 요소 추출 → Document
```

## 클래스 메서드

### `extensions()`

지원하는 파일 확장자 목록을 반환해요.

```python
@staticmethod
def extensions() -> list[str]
```

<ResponseField name="return" type="list[str]">
  지원하는 확장자 리스트. HWPX 파서의 경우 `[".hwpx"]`를 반환해요.
</ResponseField>

**사용 예시**

```python
from ureca_document_parser.hwpx import HwpxParser

print(HwpxParser.extensions())  # [".hwpx"]
```

### `parse()`

HWPX 파일을 파싱하여 `Document` 객체를 반환해요.

```python
@staticmethod
def parse(filepath: Path | str) -> Document
```

<ParamField path="filepath" type="Path | str" required>
  파싱할 HWPX 파일의 경로. 문자열 또는 `pathlib.Path` 객체를 받아요.
</ParamField>

<ResponseField name="return" type="Document">
  파싱된 문서 객체. `elements` 리스트에 `Paragraph`, `Table` 등의 문서 요소가 포함돼요.
</ResponseField>

**예외**

- `ParseError` — 파일을 찾을 수 없거나 유효한 HWPX 파일이 아닌 경우 발생해요.

**사용 예시**

```python
from ureca_document_parser.hwpx import HwpxParser
from pathlib import Path

# 문자열 경로 사용
doc = HwpxParser.parse("보고서.hwpx")

# Path 객체 사용
path = Path("제안서.hwpx")
doc = HwpxParser.parse(path)

# 문서 요소 접근
for element in doc.elements:
    if isinstance(element, Paragraph):
        print(f"[레벨 {element.heading_level}] {element.text}")
```

## Parser Protocol 구현

`HwpxParser`는 `Parser` 프로토콜을 구현해요. 상속 없이 정적 메서드 시그니처만 맞추면 되는 구조적 서브타이핑(structural subtyping) 방식이에요.

```python
class Parser(Protocol):
    @staticmethod
    def extensions() -> list[str]:
        """지원하는 파일 확장자 리스트"""
        ...

    @staticmethod
    def parse(filepath: Path | str) -> Document:
        """파일을 파싱하여 Document 반환"""
        ...
```

## 레지스트리 등록

`HwpxParser`는 자동으로 `FormatRegistry`에 등록돼요. 수동 등록이 필요한 경우:

```python
from ureca_document_parser import get_registry
from ureca_document_parser.hwpx import HwpxParser

registry = get_registry()
registry.register_parser(HwpxParser)
```

등록 후에는 확장자 기반 자동 라우팅이 가능해요:

```python
from ureca_document_parser import get_registry

registry = get_registry()
doc = registry.parse("문서.hwpx")  # HwpxParser가 자동으로 선택됨
```

## 내부 구현 세부사항

### 섹션 파일 탐색

HWPX 아카이브에서 섹션 XML 파일을 찾는 두 가지 방법을 사용해요:

1. **매니페스트 기반**: `Contents/content.hpf` 파일에서 섹션 파일 목록과 순서를 읽어요.
2. **패턴 기반 폴백**: 매니페스트가 없으면 파일명에 "section"이 포함된 XML 파일을 찾아요.

### XML 네임스페이스 처리

XML 요소를 파싱할 때 네임스페이스를 무시하고 로컬 이름만 비교해요. 다양한 HWPX 변형 포맷을 유연하게 처리하기 위한 설계예요.

```python
# '{http://www.hancom.co.kr/hwpml}p' → 'p'
```

### 제목 레벨 감지

문단의 제목 레벨을 감지하는 두 가지 방법을 순차적으로 시도해요:

1. **outlineLevel 속성**: `<pPr outlineLevel="1">` 같은 명시적 속성 확인
2. **스타일 패턴**: `styleIDRef`에서 "heading", "제목" 등의 패턴 매칭

### 중첩 테이블 지원

HWPX는 테이블 셀 내부에 다른 테이블이 중첩될 수 있어요. `TableCell.content`는 `Paragraph`와 `Table`을 모두 포함할 수 있는 리스트로 설계되어 있어요.

```python
for row in table.rows:
    for cell in row.cells:
        for item in cell.content:
            if isinstance(item, Table):  # 중첩 테이블
                process_nested_table(item)
```

### 텍스트 추출 전략

XML 구조에서 텍스트를 추출할 때 여러 패턴을 처리해요:

- `<run><t>텍스트</t></run>` — 표준 패턴
- `<p><t>텍스트</t></p>` — 일부 HWPX 변형
- `<subList><p>...</p></subList>` — 리스트 아이템
