## 함수 시그니처

```python
def convert(
    input_path: str | Path,
    output_path: str | Path | None = None,
    *,
    format: str = "markdown",
    chunks: bool = False,
    chunk_size: int = 1000,
    chunk_overlap: int = 200,
) -> str | list[LCDocument] | None
```

## 설명

입력 파일을 지정한 포맷으로 변환해요. 파일에 저장하거나, 문자열로 반환하거나, LangChain Document 청크로 반환할 수 있어요.

파일 확장자를 기반으로 적절한 파서가 자동으로 선택되며, 내부적으로 [FormatRegistry](/api/registry)를 사용해요.

## 파라미터

<ParamField path="input_path" type="str | Path" required>
  변환할 입력 파일 경로예요. `.hwp`, `.hwpx`, `.pdf` 등 지원되는 포맷이어야 해요.
</ParamField>

<ParamField path="output_path" type="str | Path | None" default="None">
  출력 파일 경로예요. 
  - `None`이면 문자열로 반환해요
  - 경로를 지정하면 해당 파일에 저장하고 `None`을 반환해요
  - `chunks=True`일 때는 무시돼요
</ParamField>

<ParamField path="format" type="str" default="markdown">
  출력 포맷 이름이예요. 현재는 `"markdown"`만 지원해요.
</ParamField>

<ParamField path="chunks" type="bool" default="False">
  `True`로 설정하면 LangChain Document 리스트로 반환해요. RAG 시스템에 바로 사용할 수 있어요.
  
  <Note>
  이 기능을 사용하려면 `langchain` extra가 필요해요:
  ```bash
  pip install ureca_document_parser[langchain]
  ```
  </Note>
</ParamField>

<ParamField path="chunk_size" type="int" default="1000">
  청크의 최대 크기(문자 수)예요. `chunks=True`일 때만 사용돼요.
</ParamField>

<ParamField path="chunk_overlap" type="int" default="200">
  청크 간 오버랩 크기(문자 수)예요. `chunks=True`일 때만 사용돼요. 컨텍스트 연속성을 유지하는 데 도움이 돼요.
</ParamField>

## 반환값

<ResponseField name="return" type="str | list[LCDocument] | None">
  반환 타입은 파라미터 설정에 따라 달라져요:
  
  <Expandable title="반환 타입별 설명">
    <ResponseField name="chunks=True" type="list[LCDocument]">
      LangChain Document 객체의 리스트예요. 각 Document는 청크된 텍스트와 메타데이터를 포함해요.
      
      메타데이터 필드:
      - `source`: 입력 파일 경로
      - `format`: 원본 파일 포맷 (예: "hwp", "hwpx")
    </ResponseField>
    
    <ResponseField name="chunks=False, output_path=None" type="str">
      변환된 Markdown 문자열이예요.
    </ResponseField>
    
    <ResponseField name="chunks=False, output_path 지정" type="None">
      파일에 저장되고 `None`을 반환해요.
    </ResponseField>
  </Expandable>
</ResponseField>

## 예외

<ResponseField name="ValueError" type="Exception">
  지원하지 않는 파일 포맷이거나 출력 포맷일 때 발생해요.
</ResponseField>

<ResponseField name="ImportError" type="Exception">
  `chunks=True`인데 `langchain` extra가 설치되지 않았을 때 발생해요.
</ResponseField>

<ResponseField name="ParseError" type="Exception">
  파일 파싱 중 오류가 발생했을 때 발생해요. 자세한 내용은 [ParseError](/api/models#parseerror)를 참고하세요.
</ResponseField>

## 사용 예시

<CodeGroup>

```python 파일에 저장
from ureca_document_parser import convert

# HWP 파일을 Markdown으로 변환하여 저장
convert("보고서.hwp", "보고서.md")

# 출력 디렉토리가 없으면 자동으로 생성됨
convert("제안서.hwpx", "output/제안서.md")
```

```python 문자열로 반환
from ureca_document_parser import convert

# Markdown 문자열로 반환
markdown = convert("문서.hwp")
print(markdown)

# 다른 처리를 위해 문자열 사용
lines = markdown.split("\n")
```

```python LangChain 청크
from ureca_document_parser import convert

# RAG용 청크로 변환
chunks = convert("보고서.hwp", chunks=True)

for chunk in chunks:
    print(f"Content: {chunk.page_content[:100]}...")
    print(f"Metadata: {chunk.metadata}")
```

```python 커스텀 청크 설정
from ureca_document_parser import convert

# 작은 청크 크기로 변환 (더 세밀한 검색용)
chunks = convert(
    "대용량_보고서.hwp",
    chunks=True,
    chunk_size=500,
    chunk_overlap=100
)

print(f"생성된 청크 수: {len(chunks)}")
```

```python 에러 처리
from ureca_document_parser import convert, ParseError

try:
    result = convert("문서.hwp", "output.md")
except ValueError as e:
    print(f"지원하지 않는 포맷: {e}")
except ParseError as e:
    print(f"파싱 오류: {e}")
except ImportError as e:
    print(f"의존성 누락: {e}")
```

</CodeGroup>

## 내부 동작

`convert()` 함수는 내부적으로 다음 단계를 거쳐요:

<Steps>
  <Step title="레지스트리 가져오기">
    [get_registry()](/api/registry#get_registry)를 호출하여 전역 FormatRegistry를 가져와요.
  </Step>
  
  <Step title="파일 파싱">
    입력 파일 확장자에 맞는 파서를 선택하여 [Document](/api/models#document) 객체로 파싱해요.
  </Step>
  
  <Step title="출력 변환">
    지정한 포맷의 Writer를 사용하여 Document를 출력 문자열로 변환해요.
  </Step>
  
  <Step title="후처리">
    - `chunks=True`: RecursiveCharacterTextSplitter로 청크 분할
    - `output_path` 지정: 파일에 저장
    - 그 외: 문자열 반환
  </Step>
</Steps>

## 참고사항

<Note>
출력 파일 경로의 부모 디렉토리가 없으면 자동으로 생성돼요 (`parents=True`).
</Note>

<Warning>
`chunks=True`를 사용하려면 반드시 `langchain` extra를 설치해야 해요:
```bash
pip install ureca_document_parser[langchain]
```
</Warning>

<Tip>
대용량 문서를 처리할 때는 `chunk_size`와 `chunk_overlap`을 조정하여 최적의 성능을 찾아보세요. 일반적으로:
- **긴 문서 검색**: chunk_size=1000-1500
- **정밀한 검색**: chunk_size=500-800
- **컨텍스트 유지**: chunk_overlap=100-200
</Tip>
